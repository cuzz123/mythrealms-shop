# MythRealms 小云雀镜头合同 V1

## 项目总规则

- 项目编号：`MR-TT-PEARL-V1`
- 成片规格：竖版 `9:16`；每次只生成一条 `4-6 秒` 镜头。
- 生成模式：图生视频的定帧延展模式。上传的佩戴关键帧既是首帧，也是人物、首饰、服装、动作、构图与场景的唯一基准。
- 每条镜头只安排一个肉眼可见的动作，一个固定机位，以及一个明确的结束状态。
- 钩子文案、产品名、价格、字幕、音乐和行动号召全部留到剪辑阶段添加。不要要求模型生成任何文字。
- 先上传佩戴关键帧，让小云雀自动插入参考图标签，再原样保留该标签。下方示例使用 `@Image1`；若界面自动生成中文或其他本地化标签，以界面生成的标签为准，不要手动改名。

## 参考图使用规则

| 资产 | 默认用途 | 上传规则 |
| --- | --- | --- |
| 佩戴关键帧 | 首帧、模特身份、产品身份、场景、姿势 | 必传，且必须第一个上传。 |
| 干净的产品标准图 | 仅校正产品结构 | 仅在产品变形后补传；不要使用带水印的供应商图。 |
| 人物身份卡 | 仅校正脸、发型、肤质、服装连续性 | 仅在人脸漂移后补传。 |
| 场景底图 | 仅校正环境 | 仅在场景漂移后补传。 |

首次生成始终只用佩戴关键帧。额外参考图会分散模型注意力并可能互相冲突；只有出现明确问题时，才补一张能修复该问题的参考图。

## 开跑前检查

1. 选择 `4-6 秒` 与 `9:16`。
2. 将对应的佩戴关键帧作为第一张、也是唯一一张参考图上传。
3. 不启用大幅镜头运动、换场、换装或生成文字。
4. 只保留环境音；最终音乐与英文文案在剪辑软件里添加。
5. 同一提示词连续生成 3 条草稿，先选出最佳版本，再决定是否改提示词。

## 质检评分卡

每条生成结果在进入下一镜头前都要评分。总分达到 `8/10`，且无致命问题，才可通过。

| 检查项 | 权重 | 通过条件 |
| --- | ---: | --- |
| 首饰结构 | 4 | 形状、数量、连接方式与材质均匹配关键帧。 |
| 人物连续性 | 2 | 脸部、发型、肤质与服装没有漂移。 |
| 手部与耳朵结构 | 2 | 没有多指、手部融合或耳朵变形。 |
| 动作与镜头 | 1 | 一个小动作完整收住；没有跳帧或镜头重置。 |
| 场景与光线 | 1 | 环境稳定；光线符合真实物理逻辑。 |

**致命问题：** 产品被重新设计、首饰脱落、脸部变化、手部畸形、产品不可辨识、生成文字，或画面切场。

## 返工规则

每条镜头最多使用 5 次生成额度：先做 3 条相同提示词草稿，随后最多做 2 次受控返工。每次返工只能改一个变量。

| 问题 | 第一次返工方式 | 不要这样做 |
| --- | --- | --- |
| 脸部或发型漂移 | 增加匹配的人物身份卡作为 `@Image2`，仅用于锁定人物身份。 | 不要同时增加新场景或新服装描述。 |
| 产品弯曲、变大或缺少部件 | 镜头保持固定；动作幅度减半；仅为产品结构补一张干净标准产品图。 | 不要同时要求转身、推近和手部动作。 |
| 手或手腕异常 | 删除手腕旋转；手保持静止，只让光线或发丝产生微动。 | 不要反复尝试同一个复杂手部动作。 |
| 耳环脱落或耳朵改变 | 把转头缩减为眼神移动，并明确写出保持耳朵结构。 | 不要增加环绕镜头。 |
| 背景改变 | 删除所有场景描述，只保留“严格保持背景布局不变”。 | 不要增加更多氛围关键词。 |
| 镜头跳动 | 将全部镜头描述替换为“镜头始终固定”。 | 不要混用手持、推近和横移。 |
| 镜头太静 | 只增加一个身体微动作和一个环境微动。 | 不要再增加第二个身体动作。 |

同一个问题连续两次没有解决，就重写出错的那一句提示词或简化动作。不要把第三次返工押在运气上。
