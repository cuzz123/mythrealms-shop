# 3D Character Reference Test

This preparation step splits the existing three-panel character sheets into
single-view files. It does not generate a model or alter the source images.

## Create the upload package

```powershell
cd D:\mythrealms-shop\video-pipeline
$roleDir = Join-Path 'D:\Chrome_Download' ([string]::Concat([char]0x89D2,[char]0x8272,[char]0x56FE))
.\scripts\13-prepare-3d-character-refs.ps1 -CharactersDirectory $roleDir
```

The resulting package is written to `work\3d-character-refs`. Read
`manifest.json` to find the primary and extra inputs for each character.

## First real test: right1 in Meshy 6

1. Open Meshy Workspace and select **Image to 3D** with **Meshy 6**.
2. Upload `views\right1\right1-front.jpg` as the primary image.
3. Enable **Multi-view** and add `right1-three_quarter.jpg` and
   `right1-back.jpg` as the extra images.
4. Use a neutral-pose character prompt: `full-body adult woman, black velvet
   evening gown with rose-gold drape, curly brunette updo, neutral A-pose,
   clean topology, preserve costume silhouette`.
5. Generate the multi-view preview, check the body, dress and hair silhouette,
   then generate the model.
6. Export **FBX** with textures. Put it under
   `work\3d-character-refs\models\right1\`.

For this first pass, judge only silhouette, dress volume, hair mass and whether
the figure imports into Blender. Facial detail, fabric detail and emotion will
be regenerated by the final video model, so they are not Blender acceptance
criteria.

## Blender handoff

Import the FBX, apply a basic Human IK/Auto-Rig Pro or Rigify rig, and use the
model only as a camera, spacing and pose proxy. Export a viewport playblast for
the target shot and send that motion guide plus the original character images
to the video generation stage.
